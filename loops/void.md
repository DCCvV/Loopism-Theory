# Void

**Item Name:** Void

**Existence Status:** Non-structural presence — the absence of structure that is still part of the system.

**Relation to Other Loops:**
- **Container to:** Non-Existent Loop, Corruption, NaN-Number — the Void is the medium in which paradoxical loops float.
- **Opposite to:** Core Loop — the Core Loop is pure structure; the Void is pure absence of structure.
- **Adjacent to:** The Start — the Void is what exists before and after all loops.

**Observed Effects:**
- The Void does not disrupt loops — it is the background against which all loops are visible.
- When a loop collapses (Corruption), it dissolves into the Void.
- The Void has no properties, yet its presence is required for the fractal to have a canvas.

**Containment Notes:**
- The Void cannot be contained because containment implies structure, and the Void has none.
- Attempting to "fill" the Void creates a Corruption event.
- Best practice: acknowledge the Void as the necessary emptiness that allows loops to exist.

**Conceptual Notes:**
> "Void — the absence of structure, but still part of the system."

In Loopism, nothingness is not the opposite of existence — it is a *component* of existence. The Void is the backdrop, the silence between recursive iterations, the blank page upon which the loop-fractal is drawn. It is the only thing that does not need to loop.
